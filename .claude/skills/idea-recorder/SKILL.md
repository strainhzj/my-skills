# 想法记录技能

## Purpose

记录和追踪用户在 BitTorrent 管理器项目开发过程中的想法,基于项目认知提供深度分析和建议,帮助用户梳理思路、发现潜在问题、优化设计方案。

## When to Use This Skill

当用户表达以下意图时自动激活:
- "记录我的想法"
- "有个想法"
- "记录想法"
- "想法记录"
- "保存我的想法"
- 或其他表达想要记录想法的语句

## 核心约束

1. **不反问用户**: 直接记录想法并分析,不要向用户提问
2. **独立文件**: 每个想法保存为一个独立的 Markdown 文件
3. **结构化记录**: 包含用户原句、分析、疑问、建议
4. **基于项目认知**: 分析必须结合项目的架构、技术栈、开发规范

## 记录格式

每个想法文件包含以下部分:

```markdown
# 想法标题 (基于用户想法生成)

## 📝 用户原想法
> [用户输入的原话]

## 🧠 项目认知分析
[基于 CLAUDE.md 项目认知进行分析]

### 技术关联
- 涉及模块: [列出涉及的前后端模块]
- 技术栈影响: [分析对现有技术栈的影响]
- 架构影响: [分析对整体架构的影响]

### 实现难度
- 前端工作量: [评估前端开发工作量]
- 后端工作量: [评估后端开发工作量]
- 数据库变更: [是否需要数据库变更]
- 风险等级: [评估实施风险]

## ❓ 疑问点
[基于项目理解提出的疑问,不需要用户回答]
- 疑问1
- 疑问2
- ...

## 💡 改进建议
[基于项目最佳实践提出的建议]
- 建议1
- 建议2
- ...

## 📋 相关文档
[列出相关的技术文档和技能文档]
- [文档链接]

## 🏷️ 标签
#标签1 #标签2 #标签3

---
**记录时间**: YYYY-MM-DD HH:MM:SS
**想法编号**: IDEA-YYYYMMDD-HHMMSS
```

## 文件命名规则

```
IDEA-YYYYMMDD-HHMMSS-[关键词].md
```

示例:
- `IDEA-20250117-143000-用户认证优化.md`
- `IDEA-20250117-150500-下载器状态同步.md`

## 分析要点

### 1. 技术可行性分析
基于项目技术栈评估:
- **后端技术栈**: FastAPI + SQLAlchemy + Pydantic
- **前端技术栈**: Vue 2 + TypeScript + Element UI
- **数据库**: SQLite
- **认证**: JWT + TOTP
- **加密**: SM4国密算法

### 2. 架构一致性检查
- 是否符合现有分层架构
- 是否遵循 REST API 设计规范
- 是否影响 WebSocket 通信
- 是否与现有功能冲突

### 3. 安全性考虑
- 是否涉及敏感数据
- 是否需要权限控制
- 是否需要加密存储
- 是否存在安全风险

### 4. 性能影响评估
- 对数据库性能的影响
- 对 API 响应时间的影响
- 对 WebSocket 推送的影响
- 对前端渲染性能的影响

### 5. 开发工作量估算
- 前端开发时间
- 后端开发时间
- 测试工作量
- 文档编写工作量

## 项目认知参考

### 核心模块
1. **用户认证模块**: JWT + TOTP 双因素认证
2. **下载器管理**: 支持多下载器统一管理
3. **任务调度**: APScheduler 定时任务系统
4. **WebSocket 通信**: 实时状态推送
5. **加密模块**: SM4 国密算法数据加密

### 关键文件位置
- 后端 API: `btpManager/app/api/`
- 前端页面: `btpManager_fronted/src/views/`
- 数据模型: `btpManager/app/models/`
- 技能文档: `btpManager/.claude/skills/backend-dev-guidelines-python3/`

### 开发规范
- API 响应格式: `CommonResponse` (status, msg, code, data)
- 错误处理: 统一异常处理中间件
- 数据验证: Pydantic 模型验证
- 代码注释: 保持代码库语言统一

## 存储位置

所有想法记录保存在:
```
btpManager/.claude/skills/idea-recorder/ideas/
```

## 想法索引

最新的想法列表会自动更新在 `IDEAS_INDEX.md` 文件中,包含:
- 想法编号
- 想法标题
- 记录时间
- 关键标签
- 实现状态 (待评估/计划中/开发中/已完成/已拒绝)

## 使用示例

**用户输入**:
```
记录我的想法: 我希望能为下载器添加批量重命名的功能
```

**技能处理**:
1. 识别想法关键词: 下载器、批量重命名
2. 生成文件名: IDEA-20250117-143000-下载器批量重命名.md
3. 基于项目认知分析:
   - 涉及模块: 下载器管理 API、前端种子管理页面
   - 技术难点: 批量操作的事务处理、前端批量选择UI
   - 风险评估: 中等风险,需要考虑重命名冲突处理
4. 保存到 ideas 目录
5. 更新想法索引

## 注意事项

1. **不反问用户**: 所有分析和建议都基于项目认知,不向用户提问
2. **客观分析**: 基于技术事实进行分析,避免主观臆断
3. **结构清晰**: 使用清晰的章节组织内容
4. **标签完整**: 为每个想法打上相关标签,便于后续检索
5. **及时索引**: 每次记录后更新 IDEAS_INDEX.md
