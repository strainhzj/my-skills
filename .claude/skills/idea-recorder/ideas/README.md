# 想法记录目录

本目录用于存储所有用户想法记录,每个想法保存为独立的 Markdown 文件。

## 文件命名规范

```
IDEA-YYYYMMDD-HHMMSS-[关键词].md
```

示例:
- `IDEA-20250117-143000-用户认证优化.md`
- `IDEA-20250117-150500-下载器状态同步.md`

## 目录结构

```
ideas/
├── README.md                    # 本说明文件
├── IDEAS_INDEX.md              # 想法索引(自动更新)
├── IDEA-20250117-143000-*.md   # 具体想法文件
└── ...
```

## 想法文件格式

每个想法文件包含:
- 用户原想法
- 项目认知分析
- 技术关联
- 实现难度评估
- 疑问点
- 改进建议
- 相关文档链接
- 标签

## 想法状态

每个想法有以下状态:
- **待评估**: 新记录的想法,尚未评估
- **计划中**: 已评估,计划实施
- **开发中**: 正在开发实现
- **已完成**: 功能已实现
- **已拒绝**: 经评估后决定不实施
- **暂缓**: 暂时搁置,可能将来实施

## 维护说明

1. **自动更新**: 想法索引 (IDEAS_INDEX.md) 会在每次添加新想法时自动更新
2. **状态变更**: 想法状态变更时,需要同步更新 IDEAS_INDEX.md
3. **定期审查**: 建议定期审查想法列表,更新状态和优先级

## 检索方式

1. **按标签**: 在 IDEAS_INDEX.md 中查看标签云
2. **按状态**: 筛选特定状态的想法
3. **按时间**: 按记录时间排序
4. **关键词**: 使用文件名关键词搜索

---

**注意**: 请不要手动修改 IDEA 编号和时间戳,保持系统的有序性。
